# captcha

Репозиторий с набором server-driven CAPTCHA для sandbox и SDK.

## Требования к капчам

### Пользовательские

1. Должна быть максимально простой и понятной для человека.
2. Очень желательно взаимодействие вроде перетаскивания, кнопок и свайпов, так как простые сценарии давно генерируются.
3. Как вариант, допустим очень простой аналог онлайн-игр, когда с клиента идет стрим событий, а с сервера стрим данных для рендера.
4. Все надписи на русском языке.

### Технические

1. Задание генерируется и отдается клиенту в рантайме, чтобы нельзя было атаковать систему по заранее известному набору задач.
2. Требование по производительности: минимум `100 RPS` на генерацию с одного инстанса на современных CPU.
3. Желательна экономия памяти: задание сгенерировали и отдали клиенту, на сервере сохраняется только минимальное состояние для проверки.
4. Для игровых сценариев серверное состояние допустимо, но память все равно нужно экономить. Целевое потребление на `10k` активных задач в процессе: до `8 GB`.
5. Значения `complexity` и `confidence` должны быть в диапазоне `1..100`.
6. Один клик или жест должен означать одно событие, уходящее на сервер.

## Документирование

1. Необходимо предоставить базовые описания механик.
2. Необходимо задокументировать влияние уровней сложности (`complexity`) на поведение.

## Сведения об интеграции

### Подключение к системе

Со стороны backend все завернуто в интерфейс SDK.

Пример взаимодействия с клиентской частью CAPTCHA находится в `captcha/captcha-example/internal/captcha/captcha.go`.

Важно:

- метод `Stream` нужен только для серверного стрима данных, которые должны приходить клиенту в процессе решения;
- результат challenge нужно посылать в тот же стрим, куда приходит событие в `HandleFrontendEvent`;
- `HandleConnectionClosed` и `HandleCloseChallenge` должны очищать состояние challenge из памяти;
- `HandleBalancerEvent` на текущем этапе можно игнорировать.

### Работа CAPTCHA со стороны клиента

Challenge представляет собой сообщение с `id` и `html`, который запускается на фронте в `iframe`.

HTML должен содержать все целиком, включая `js/css`.

В `captcha/captcha-example/frontend` лежит минимальный пример на `vite/ts/scss` для разработки фронта. На выходе он собирается в один файл.

Отправку и прием данных смотреть в `captcha/captcha-example/frontend/src/shared/app.ts`.

Данные отправляются в байтах, поэтому формат может быть любым, но он должен быть компактным. Для событий желательно использовать бинарное представление.

## Как использовать песочницу

По умолчанию при запуске `./bin/sandbox` либо через `docker compose` песочница слушает нужные порты и подключается к CAPTCHA-инстансам.

После подключения CAPTCHA нужно нажать `Refresh` в интерфейсе, и тогда появится их список.

В интерфейсе можно выбрать нужные сложность и точность.

## Известные проблемы

1. Возможны небольшие погрешности интерфейса, их стоит учитывать при тестировании.
